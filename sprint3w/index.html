    Administració de dominis amb Windows Server

    1. Fonaments de la configuració de xarxa

    Teoria

    En una xarxa d’ordinadors, cada dispositiu necessita una adreça IP per comunicar-se. Si un servidor utilitza una IP dinàmica (assignada per DHCP), aquesta pot canviar i provocar problemes d’accés als serveis. Per això, assignar una IP estàtica al servidor garanteix que sempre serà localitzable a la mateixa adreça, millorant l’estabilitat i la seguretat de la infraestructura.

    Assignar una IP estàtica

    El primer que s'ha de fer a un servidor és configurar una IP estàtica per evitar que el servidor canviï automàticament la seva IP per DHCP, cosa que podria provocar desconfiguracions.

    2. Instal·lació i configuració d'Active Directory

    Teoria

    Active Directory (AD) és una eina centralitzada de Microsoft per gestionar usuaris, equips i altres recursos dins d’una xarxa. Permet establir polítiques de seguretat, controlar l’accés i simplificar la gestió de grans volums d’usuaris i dispositius. L’AD funciona a través d’un o més controladors de domini, servidors especialitzats que mantenen la base de dades d’identitats i permisos.

    2.1. Server

    Afegim rols i característiques.

    Seguim amb la configuració per defecte, excepte en aquelles pantalles on sigui necessari fer modificacions (s'indiquen a continuació).

    És molt important comprovar que la IP del servidor sigui la que hem assignat prèviament.

    Afegim la característica de "Servicio de dominio de Active Directory".

    Instal·lem.

    Quan acabi, a la bandereta apareixerà un triangle d'advertència. Fes clic i selecciona "Promover este servidor a controlador de dominio".

    Afegim un nou bosc i un nou nom de domini.

    Assignem una contrasenya.

    Fem clic a "Següent" i esperem fins arribar a la finestra d'instal·lació. Fes clic a "Instalar" i espera que acabi.

    En acabar, en fer clic a "Cerrar", el servidor es reiniciarà automàticament per aplicar els canvis.

    3. Usuaris i equips d'Active Directory

    Teoria

    Centralitzar la gestió d’usuaris a través d’Active Directory permet administrar de manera eficient permisos, grups i polítiques. Això facilita l’accés controlat als recursos, la seguretat i l’automatització de tasques administratives.

    Per poder gestionar usuaris des de qualsevol equip, primer hem de crear-los a "Usuaris i equips d'Active Directory".

    Un cop dins, anem a "Users" i creem un nou usuari.

    Assignem la contrasenya.

    3.1. Client

    Teoria

    Quan un ordinador es connecta a un domini, passa a ser gestionat pel controlador de domini. Això permet aplicar polítiques de seguretat, desplegar programari i gestionar permisos de manera centralitzada.

    Al client, el primer que fem és configurar l'adaptador d'ethernet, posant al DNS la IP del servidor.

    Comprovem la connexió fent ping al servidor.

    Anem a "Configuración -> Información del Sistema" i canviem la configuració.

    Escrivim el nostre nom de domini.

    Introduïm el nom d'usuari administrador del servidor i la contrasenya.

    Reiniciem l'equip.

    Comprovem que l'equip s'ha connectat correctament al servidor.

    4. Gestió de Domini: Activitats Pràctiques i Conceptes

    Teoria

    Les unitats organitzatives (OU) i els grups permeten estructurar i organitzar els recursos dins del domini. Les polítiques de grup (GPO) serveixen per aplicar configuracions i restriccions automàticament als equips i usuaris del domini. La gestió centralitzada d’aquests elements facilita el control, la seguretat i l’eficiència de l’administració.
